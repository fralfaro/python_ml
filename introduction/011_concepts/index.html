    Representaci√≥n de Datos¬∂

    Una tabla es una cuadr√≠cula bidimensional de datos, en la que las filas representan elementos individuales del conjunto de datos y las columnas representan cantidades relacionadas con cada uno de estos elementos.

    Por ejemplo, consideremos el conjunto de datos Iris, analizado por Ronald Fisher en 1936.

    De manera similar, cada columna de los datos se refiere a una pieza particular de informaci√≥n que describe cada muestra. En general, nos referiremos a las columnas de la matriz como caracter√≠sticas (features), y el n√∫mero de columnas como n_features.

    Matriz de caracter√≠sticas¬∂

    La matriz de caracter√≠sticas (features matrix en ingl√©s) es una representaci√≥n de los datos de entrada en un problema de aprendizaje autom√°tico. Es una matriz bidimensional (con forma [n_samples, n_features]), en la que las filas representan las muestras o instancias de los datos, y las columnas representan las caracter√≠sticas o atributos que describen cada muestra.

    Las caracter√≠sticas (es decir, columnas) siempre se refieren a las observaciones distintas que describen cada muestra de manera cuantitativa. Las caracter√≠sticas generalmente tienen valores reales, pero en algunos casos pueden ser valores booleanos o discretos.

    Por convenci√≥n, esta matriz de caracter√≠sticas a menudo se almacena en una variable llamada X.

    Conjunto de Entrenamiento y Prueba¬∂

    El conjunto de entrenamiento (train set) y el conjunto de prueba (test set) son dos conjuntos de datos distintos que se utilizan en el aprendizaje autom√°tico para entrenar y evaluar un modelo.

    Definici√≥n¬∂

    • El conjunto de entrenamiento es un subconjunto de los datos de entrada que se utiliza para ajustar los par√°metros del modelo y encontrar la mejor funci√≥n que pueda predecir la variable de salida a partir de las variables de entrada. En otras palabras, se utiliza para entrenar el modelo.

    • El conjunto de prueba es otro subconjunto de los datos de entrada que se utiliza para evaluar el rendimiento del modelo. Este conjunto de datos no se utiliza en el entrenamiento del modelo, sino que se utiliza √∫nicamente para evaluar el rendimiento del modelo en datos nuevos y no vistos durante el entrenamiento.

    La funci√≥n train_test_split de Scikit-learn es una herramienta √∫til para dividir un conjunto de datos en conjuntos de entrenamiento y prueba.

    La funci√≥n train_test_split toma como entrada los datos que se van a dividir, junto con los valores correspondientes de la variable de respuesta. La funci√≥n tambi√©n permite ajustar la proporci√≥n de datos que se asignar√°n al conjunto de entrenamiento y al conjunto de prueba. Por defecto, la proporci√≥n es del 75% para el conjunto de entrenamiento y el 25% para el conjunto de prueba.

    Para esto, podemos utilizar la funci√≥n train_test_split:

    Motivaci√≥n¬∂

    La raz√≥n de utilizar dos conjuntos de datos separados para entrenar y evaluar el modelo es evitar el sobreajuste (overfitting) del modelo. El sobreajuste ocurre cuando un modelo se ajusta demasiado bien a los datos de entrenamiento y no generaliza bien a nuevos datos. Si se utilizan los mismos datos para entrenar y evaluar el modelo, es probable que el modelo tenga un buen rendimiento en los datos de entrenamiento pero un rendimiento pobre en datos nuevos.

    Reglas de separaci√≥n¬∂

    En general, se divide el conjunto de datos de entrada en dos conjuntos: el conjunto de entrenamiento y el conjunto de prueba. El conjunto de entrenamiento suele ser el 70-80% del conjunto de datos y el conjunto de prueba el 20-30%. La proporci√≥n puede variar seg√∫n el tama√±o del conjunto de datos y la complejidad del modelo.

    A modo de regla emp√≠rica, se considerar√° el tama√±o √≥ptimo basado en la siguiente tabla:

    n√∫mero de filas train set test set
    entre 100-1000 67% 33%
    entre 1.000- 100.000 80% 20%
    mayor a 100.000 99% 1%

    M√°s informaci√≥n¬∂

    Es importante destacar que, en algunos casos, se utiliza un tercer conjunto de datos, llamado conjunto de validaci√≥n (validation set), que se utiliza para ajustar los hiperpar√°metros del modelo y evitar el sobreajuste. En este caso,

    • el conjunto de entrenamiento se utiliza para ajustar los par√°metros del modelo,
    • el conjunto de validaci√≥n se utiliza para ajustar los hiperpar√°metros
    • y el conjunto de prueba se utiliza para evaluar el rendimiento final del modelo.

    Error de estimaci√≥n¬∂

    El error de estimaci√≥n se refiere a la diferencia entre el valor real de una variable y el valor estimado por un modelo. En el contexto de los modelos de aprendizaje autom√°tico, el error de estimaci√≥n se refiere a la diferencia entre la variable objetivo (tambi√©n conocida como variable dependiente o variable de respuesta) y la variable predicha por el modelo.

    El objetivo de los modelos de aprendizaje autom√°tico es minimizar el error de estimaci√≥n y, por lo tanto, mejorar la precisi√≥n del modelo en la predicci√≥n de valores desconocidos. El error de estimaci√≥n se puede calcular utilizando diferentes m√©tricas seg√∫n el tipo de modelo y el objetivo del an√°lisis.

    Para medir el error de estimaci√≥n en machine learning es necesario identificar el tipo de dato del target: num√©ricos o categ√≥ricos.

    Error de estimaci√≥n: Datos Num√©ricos¬∂

    En este caso, el error de estimaci√≥n corresponde a la diferencia entre el valor original y el valor predicho,es decir:

    $$e_{i}=y_{i}-\hat{y}_{i} $$

    Este resultado nos da el error de estimaci√≥n por cada muestra (o fila). Para encontrar un resultado que resuma el error de estimaci√≥n utilizaremos funciones de distancias o m√©tricas de estimaci√≥n.

    1. M√©tricas absolutas: Las m√©tricas absolutas o no escalada miden el error sin escalar los valores. Las m√©trica absolutas m√°s ocupadas son:

      • Mean Absolute Error (MAE)

      $$\textrm{MAE}(y,\hat{y}) = \dfrac{1}{n}\sum_{t=1}^{n}\left | y_{t}-\hat{y}_{t}\right |$$

      • Root Mean Squared Error (RMSE):

      $$\textrm{RMSE}(y,\hat{y}) =(\dfrac{1}{n}\sum_{t=1}^{n}\left ( y_{t}-\hat{y}_{t}\right )^2)^{1/2}$$

    2. M√©tricas Porcentuales: Las m√©tricas porcentuales o escaladas miden el error de manera escalada, es decir, se busca acotar el error entre valores de 0 a 1, donde 0 significa que el ajuste es perfecto, mientras que 1 ser√≠a un mal ajuste. Cabe destacar que muchas veces las m√©tricas porcentuales puden tener valores mayores a 1.Las m√©trica Porcentuales m√°s ocupadas son:

      • Mean absolute percentage error (MAPE):

      $$\textrm{MAPE}(y,\hat{y}) = \dfrac{1}{n}\sum_{t=1}^{n}\left | \frac{y_{t}-\hat{y}_{t}}{y_{t}} \right |$$

    Matriz de confusi√≥n

    La matriz de confusi√≥n es una herramienta que permite la visualizaci√≥n del desempe√±o de un algoritmo Para la clasificaci√≥n binaria (por ejemplo, 0 y 1), se tiene la siguiente matriz de confusi√≥n:

    Ac√° se define:

    • TP = Verdadero positivo: el modelo predijo la clase positiva correctamente, para ser una clase positiva.
    • FP = Falso positivo: el modelo predijo la clase negativa incorrectamente, para ser una clase positiva.
    • FN = Falso negativo: el modelo predijo incorrectamente que la clase positiva ser√≠a la clase negativa.
    • TN = Verdadero negativo: el modelo predijo la clase negativa correctamente, para ser la clase negativa.

    En este contexto, los valores TP Y TN muestran los valores correctos que tuve al momento de realizar la predicci√≥n, mientras que los valores de de FN Y FP denotan los valores que me equivoque de clase.

    M√©tricas de estimaci√≥n

    En este contexto, se busca maximizar el n√∫mero al m√°ximo la suma de los elementos TP Y TN, mientras que se busca disminuir la suma de los elementos de FN y FP. Para esto se definen las siguientes m√©tricas:

    1. Accuracy

    $$accuracy(y,\hat{y}) = \dfrac{TP+TN}{TP+TN+FP+FN}$$

    1. Recall:

    $$recall(y,\hat{y}) = \dfrac{TP}{TP+FN}$$

    1. Precision:

    $$precision(y,\hat{y}) = \dfrac{TP}{TP+FP} $$

    1. F-score:

    $$fscore(y,\hat{y}) = 2\times \dfrac{precision(y,\hat{y})\times recall(y,\hat{y})}{precision(y,\hat{y})+recall(y,\hat{y})} $$

    Veamos un ejemplo pr√°ctico, las m√©tricas de estimaci√≥n las podemos obtener desde sklearn.metrics:

    In¬†[14]:
    Copied! 
    from sklearn.metrics import accuracy_score, recall_score, precision_score, f1_score

accuracy = accuracy_score(df['y'],df['y_estimado'])
recall = recall_score(df['y'],df['y_estimado'])

    Curva ROC y AUC

    La curva ROC (Receiver Operating Characteristic) se utiliza para evaluar el rendimiento de los algoritmos de clasificaci√≥n binaria, es decir, entre dos clases o categor√≠as (1 o 0, Verdadero o Falso, etc.). La curva ROC proporciona una representaci√≥n gr√°fica, en lugar de un valor √∫nico como la mayor√≠a de las otras m√©tricas.

    Antes de todo, aclaremos que en la clasificaci√≥n binaria, hay cuatro resultados posibles para una predicci√≥n: Verdadero positivo (TP), Falso positivo (FP), Verdadero negativo (TN) y Falso negativo (FN).

    La curva ROC se genera calculando y trazando la tasa de verdaderos positivos (TPR) contra la tasa de falsos positivos (FPR) para un solo clasificador en una variedad de umbrales (en el siguiente p√°rrafo se definen matem√°ticamente ambas tasas). Por ejemplo, en un modelo de regresi√≥n log√≠stica, el umbral ser√≠a sobre la probabilidad predicha de una observaci√≥n perteneciente a la clase positiva. Normalmente, en la regresi√≥n log√≠stica, si se predice que una observaci√≥n ser√° positiva con una probabilidad mayor que 0.5, se etiqueta como positiva. Sin embargo, realmente podr√≠amos elegir cualquier umbral entre 0 y 1 (0.1, 0.3, 0.6, 0.99, etc.), y, de este modo, la curva ROC nos ayudar√≠a a visualizar c√≥mo estas elecciones afectan al rendimiento del clasificador.

    La tasa de verdaderos positivos (TPR), o Sensibilidad, se define como:

    $$TPR= \dfrac{TP}{TP+FN}$$

    donde TP es el n√∫mero de Verdaderos positivos y FN es el n√∫mero de Falsos negativos. La tasa de verdaderos positivos es una medida de la probabilidad de que una instancia positiva real se clasifique como positiva.

    La tasa de falsos positivos (FPR), o 1 ‚Äì Especificidad, se define como:

    $$FPR= \dfrac{FP}{FP+TN}$$

    donde FP es el n√∫mero de Falsos positivos y TN es el n√∫mero de Verdaderos negativos. La tasa de falsos positivos es esencialmente una medida de la frecuencia con la que se producir√° una ¬´falsa alarma¬ª, o la frecuencia con la que una instancia negativa real se clasificar√° como positiva.

    En la siguiente figura se muestra como ser√≠a la curva ROC para tres hipot√©ticos clasificadores distintos.

    La l√≠nea de puntos gris representa un clasificador aleatorio (equivalente a ¬´adivinar al azar¬ª); su curva ROC consiste en una l√≠nea recta diagonal de pendiente 1. Opuestamente, la l√≠nea violeta representa un clasificador perfecto, es decir, que no comete ning√∫n error (uno con una tasa de verdaderos positivos del 100% y una tasa de falsos positivos del 0%).

    Casi todos los ejemplos del mundo real se ubicar√°n en alg√∫n lugar entre estas dos l√≠neas; no es perfecto, pero proporciona m√°s poder de predicci√≥n que la suposici√≥n aleatoria. Por lo general, lo que buscamos es un clasificador que mantenga una alta tasa de verdaderos positivos y al mismo tiempo tenga una baja tasa de falsos positivos; este clasificador ideal tender√≠a a pasar muy pr√≥ximo a la esquina superior izquierda de la figura anterior, casi rozando a la l√≠nea p√∫rpura.

    El AUC

    Si bien es √∫til visualizar la curva ROC de un clasificador binario, en muchos casos podemos resumir esta informaci√≥n en una sola m√©trica: el AUC. El AUC significa √°rea bajo la curva (ROC) en ingl√©s. Generalmente, cuanto mayor es la puntuaci√≥n AUC, mejor es el rendimiento de un clasificador binario para una tarea de clasificaci√≥n dada.

    La siguiente figura muestra que para un clasificador sin poder predictivo (es decir, de predicci√≥n aleatoria), el AUC es 0.5, y para un clasificador perfecto, el AUC es 1.0; esto es el √°rea bajo sus correspondientes curvas ROC.

    La mayor√≠a de los clasificadores estar√°n entre 0.5 y 1.0, con la rara excepci√≥n de que el clasificador funcione peor que la predicci√≥n aleatoria (AUC<0.5); en este caso se estar√≠an efectuando predicciones invertidas, es decir, se tender√≠a a predecir la clase negativa cuando esta fuera realmente positiva y viceversa.

    ¬øPor qu√© usar las curvas ROC?

    • Una ventaja que presentan las curvas ROC es que nos ayudan a encontrar un umbral de clasificaci√≥n que se adapte a nuestro problema espec√≠fico.

      • Ejemplo: si estuvi√©ramos evaluando un clasificador de correo no deseado, querr√≠amos que la tasa de falsos positivos fuera realmente baja. No quer√≠amos que nadie perdiera un correo electr√≥nico importante en el filtro de spam solo porque nuestro algoritmo es demasiado agresivo. Probablemente, incluso permitir√≠amos una buena cantidad de correos electr√≥nicos no deseados reales (verdaderos positivos) a trav√©s del filtro solo para asegurarnos de que no se perdieran correos electr√≥nicos importantes.

      • Ejemplo: si nuestro clasificador estuviera prediciendo si alguien tiene una enfermedad terminal, podr√≠amos estar de acuerdo con un mayor n√∫mero de falsos positivos (diagn√≥sticos incorrectos de la enfermedad), solo para asegurarnos de que no perdamos ning√∫n positivo verdadero (personas que realmente tienen la enfermedad).

    • Adem√°s, las curvas ROC y las puntuaciones AUC tambi√©n nos permiten comparar el rendimiento de diferentes clasificadores para el mismo problema y as√≠ poder elegir el de mayor rendimiento.
